# Чек-лист тестирования

Проведены все типы тестирования
- Тестирование требований
- Чек-лист — общий поверхностный план (что проверить)
- Тест-кейс — пошаговый детальный план (как проверить)
  - Описание
  - Шаги
  - Ожидаемый результат (требование?)

Иерархия проблем?
- Ошибка (Error) — ❓. Превращается в дефект
- Дефект ( ❓ ) — ❓. Превращается в баг
- Баг (Bug) — созданных багрепорт в системе для исправления ошибок

Использованы техники
- Разбит на классы эквивалентности
- Выделены граничные значения
- Диаграмма состояний-переходов
- Утилита `Pairwise`
- Таблица принятия решений
- Причина/следствие
- Предугадывание ошибки
- Баг-репорт — главный документ который пишет тестировщик
- Пирамида-тестирования
  - Модульное (unit)
  - Интеграционное (integration)
  - Системное (system)
  - E2E (end to end, приёмочное)
